  • This course doesn’t focus on code, it’s about how to create an efficient workflow

  • The bower.json has our dependencies for our project and so these two lines of code right here tell bower, which we’re also using in this project, to actually go and download any of the libraries that we want
  • You don’t have to worry about where to put everything, it’s all taken care of with the workflow

  • Go to gruntjs.com and click on the plugin sections to look for plugins

  • If you go to the plugin sections in gruntjs.com look plugins for Sass
  • Go to terminal and install npm sass
  • Then install grunt sass plugin
  • You are going to notice that grunt contrib sass is a little bit slower
  • Now we have to automate our workflow
  • We have to grab the grunt contrib watch plugin
  • This plugin will allow us to look over our workflow and automate it
  • So with this plugin, it’s doing the processing for us as soon as we change any of our documents, whether it’s the sass documents, html document, or any of the scripts
  • If we make any changes it’s going to process our tasks and whenever we refresh our page, we can see any of the changes
  • Now we don't have to keep on running the grunt command in our terminal every time we update our project
  • Now that grunt is running our tasks automatically whenever our page changes, we still have to hit the reload button ourselves to make a change to our page
  • So we go use a livereload option to keep reloading the page
  • If you go into your terminal, every time you make a change you should see the terminal updating you on what has happened, It notices that there has been a change here on this _modules file, and then it runs all of the different scripts and automatically updates your browser.
