  • Opening, Viewing, and Editing Files
  • Line number on the left side, bottom right shows file type
  • Right Side shows the mini-map of current open file
  • Ctrl + shift + t, opens the last closed file
  • Alt 1-9 shifts through tabs
  • Ctrl + tab to go back to recent tab
  • Ctrl + p for popup palette for all tabs
  • Adjust line endings in the view tab for windows, unix, or mac
  • Right-clicking on words can spell-check a word and you can replace it too
  • Indent with Ctrl + [ or ctrl + ]
  • Ctrl + M allows you to jump between curly braces
  • Dragging a tab out allows you to view both tabs
  • New view in file, allows for same file to open in multiple tabs for viewing of different parts of the same file
  • Holding ctrl while dragging a project folder allows you to duplicate a project
  • Dragging project folder into sublime opens a sidebar for project viewing
  • You can switch projects by ctrl + alt + p
  • Ctrl + f to find word in file
  • Ctrl + i for incremental find
  • Ctrl + h for find and replace
  • Navigate search results with f4
  • Customizing Your Editor
  • Under sublime text in the top go to preferences to change to default or to your own preferences
  • Choose, add, or take away options that you want for your own preference
  • You can create different preferences for each file type
  • You can apply different preferences to different destination
  • Check out the soda theme for sublime for a nice color layout
  • Download file and rename it Theme - soda then drag it into the packages directory
  • “Theme”: “Soda Light.sublime-theme” in the custom preference to change theme
  • Make sure to restart application to see the changes
  • Check out kuroir theme online
  • Use the same instructions as the soda theme
  • Go to preferences and open key bindings - default to check out the standard key bindings
  • Don’t edit this file
  • Copy key bindings you want to edit to the user settings tab
  • Then change in the user defined file
  • Power Editing
  • Holding down alt + left or right with move through words instead of letters
  • If cursor is on word , Ctrl + d to select word
  • Ctrl+l to select the whole line
  • Ctrl + A to select contents of tag
  • Ctrl + j for brackets selection
  • Ctrl + shift + space to select scope
  • Ctrl + t to transpose to swap words
  • Ctrl + f2 to bookmark and f2 to return to bookmark
  • Using # in the goto anything allows you to incremental search through all files in the project
  • Using : and number for goto line number
  • Using @ to look through symbols
  • Ctrl +shift +p to look through all commands
  • Multiple selection, hold ctrl + click or drag
  • Alt + f3 to find all
  • Ctrl + d for class attribute , ctrl + k to skip
  • Ctrl + shift + a to select all content within tag after multiple selection
  • Copy and paste it into a new tab for easier edit ,multiple select again to paste it back in
  • Go to preferences, open setting default, copy the vintage line and paste it into the user settings file, delete the [] content to turn on vintage mode
  • Automation
  • Ctrl + spacebar to invoke the autocompletion manually
  • “auto_complete_ on_tab” is better, to get rid of autocompletion on enter
  • Hit tab for best guess and tab again to cycle through guesses
  • Press tab to cycle through the code snippet fields after autocompletion
  • Ctrl + q to start and stop macro recording
  • The macro will record your keystrokes to playback for easier coding
  • Make sure to save the macros and keybind it to make it easier to access macros
  • Essential Packages
  • Use package control to simplify packages
  • Install readmeplease first, to open readme for any package
  • Install colorpicker for fast color codes
  • Copy your line endings package and edit it in your user settings to modify, for windows
  • Edit the user sidebar settings, to customize which browser will pop up
  • Ctrl + atl+ shift+l to generate a google search hyperlink
  • Ctrl + alt + l to create a hyperlink and search the title of the site
  • Emmet, allows commands like ul#nav>li.items$*4>a{item $} to autogenerate a html code
  • Ctrl + / for commenting out the highlighted tag
  • Emmet allows ctrl + shift +/ to comment out the current block
  • Go to the sublime text packages to check out more packages
  • Conclusion
  • Read the documentation in the help tab, to know more about sublime text
  • Check out the Api reference to develop for sublime
  • Check the sublime forum to discuss and find out more about sublime text
