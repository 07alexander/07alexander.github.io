Site vs App

My understanding of a website is that it's really only meant to show you information about a business or whatever content they would like to show. Such as a restaurant posting a website to show their open hours and maybe their menus so potential clients could see the menu without actually having to go to the physical location. While a web application is a website that provides a service online like online banking website where you can check your statements or schedule payments. The web application allows user to interact with the website and manipulate data rather than only being able to see the data content. This website, stackoverflow (Links to an external site.), has an interesting topic on this subject and they mention that "a website is defined by its content, while a web application is defined by its interaction with the user". On msdn (Links to an external site.) they actually talk about the developmental processes, scenarios, and they outline the more technical side between the two.

Built It And They Will Come

I would love to build a simple web application game. Nothing too hard, just a simple card game using a standard deck of cards. I love video games and I would love to create games one day, but for now, I would like to start with just a simple card game that I used to play with my friends. I made a simple game using C++, but I wonder how developers can port or create games on a website. It looks like it would be so much more difficult than an executable file on a computer. And from what I understand, there is only a limited amount of resource the developer could draw from on a browser. It would be awesome if I could build it for my android phone too. It would also be awesome if I could make a small mock up of Facebook on a small network, just to see how that would work. Other than that, nothing else comes to mind.
